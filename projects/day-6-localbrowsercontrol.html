The Problem

Traditional browser automation has significant limitations:

  • Manual automation is time-consuming and repetitive
  • Selenium/Playwright breaks when UI changes
  • Requires coding expertise to set up
  • No visual understanding - just DOM manipulation
  • Can't adapt to unexpected scenarios

The Solution

AI-powered automation that sees and understands:

  • Vision AI interprets any interface visually
  • Self-healing adapts to UI changes automatically
  • Natural language commands - no coding required
  • Works with any application, not just browsers
  • Claude debugs and fixes its own mistakes

Core Capabilities

Screen Vision

Claude analyzes visual content on your display to understand context, read text, identify UI elements, and comprehend what's happening on screen.

Mouse Control

Automated clicking, scrolling, and dragging operations with pixel-perfect precision. Navigate any interface programmatically.

Keyboard Input

Text entry, hotkey combinations, form navigation, and keyboard shortcuts. Type naturally into any application.

Self-Healing

Autonomous debugging and error correction. When something fails, Claude diagnoses the issue and implements fixes in real-time.

100% Local

Everything runs on your machine. Your data, screenshots, and automation steps never leave your local environment.

Natural Language

Just tell Claude what to do in plain English. "Search for flights to NYC" or "Fill out this form with my info."

Setup Guide

1

Method 1: Docker (Recommended)

Quick 2-minute setup with isolated virtual Linux desktop:

docker run -p 8080:8080 anthropics/computer-use

Access the interface at localhost:8080

2

Method 2: Claude Code + Local MCP

Direct integration with your Windows machine via WSL2:

npm install -g claude-local-control
3

Install Dependencies (WSL2 Method)

Install required screenshot and input tools:

sudo apt install scrot xdotool gnome-screenshot
4

Start Automating!

Tell Claude what to do in natural language:

"Search Google Flights for NYC to LA next week"

The Self-Fixing Bug Story

During development, screenshot tools timed out in WSL2/Wayland environments. Instead of failing, Claude Code autonomously:

1. Diagnosed gnome-screenshot timeout
2. Tested alternatives (scrot)
3. Fixed prioritized scrot
4. Deployed patch in real-time

This showcases Claude's agentic capabilities - the ability to identify issues, reason about solutions, and implement fixes without human intervention.
