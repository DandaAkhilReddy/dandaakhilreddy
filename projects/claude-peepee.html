The Problem

Claude Code starts fresh every conversation. Even in the same project, it:

  • Re-reads the same files repeatedly
  • Forgets architectural decisions you explained
  • Loses track of project conventions
  • Burns tokens rediscovering context

The Solution

Claude PeePee provides persistent, project-aware memory:

  • Memories stored locally per project
  • Automatic context injection via MCP
  • 80-90% reduction in context overhead
  • Cross-session knowledge retention

Key Features

Project-Scoped Memory

Each project gets its own isolated memory store. Work on multiple projects without context bleeding.

Semantic Search

Find relevant memories using natural language queries. No need to remember exact keywords.

Zero Token Cost

Memories are injected via MCP resources, not counted against your context window.

Local & Private

All data stored locally in SQLite. Your project knowledge never leaves your machine.

Easy Integration

Single binary, simple config. Works with Claude Code out of the box via MCP.

CLI Tools

Manage memories from command line. Add, search, list, and delete memories easily.

Quick Start

1

Clone the Repository

git clone https://github.com/DandaAkhilReddy/claude_peepee.git
2

Build the Binary

cd claude_peepee && go build -o peepee ./cmd/peepee
3

Configure Claude Code

Add to your Claude Code MCP settings:

{
  "mcpServers": {
    "peepee": {
      "command": "/path/to/peepee",
      "args": ["serve"]
    }
  }
}
4

Start Using

Claude Code will now have access to persistent memory tools!

peepee add "This project uses React 18 with TypeScript"

Claude PeePee acts as an MCP server that Claude Code connects to. When you ask Claude about your project, it can query the memory store for relevant context. All communication happens via JSON-RPC over stdio.
