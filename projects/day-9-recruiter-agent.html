Recruiter Agent

AI-Powered Talent Scouting & Engagement — paste a JD, get a ranked shortlist with explainable Match & Interest scores

A demo-grade talent platform that takes a Job Description and returns a ranked candidate shortlist in ~90 seconds. The pipeline parses the JD with gpt-4o, runs a 3-stage matcher (hard SQL filter → pgvector cosine top-50 → LLM rerank with per-dimension justifications), then orchestrates simulated recruiter ↔ candidate-persona conversations streamed live to the UI via SSE. A Judge agent reads each transcript and emits an Interest Score with signals, concerns, and reasoning. A tunable Ranker combines them (default 0.6·Match + 0.4·Interest) — sliders re-weight from stored raw scores without an LLM re-call. Built on Azure with autoscale-to-0: ~$25-30/mo idle, ~$1 per JD run.

The Problem

Recruiters drown in resumes and don't trust opaque match scores:

  • A single number like "Match: 78" is impossible to interrogate
  • Keyword filters miss strong candidates and pass weak ones
  • Pure resume scoring ignores actual interest in the role
  • Outreach personalization at scale is too time-consuming
  • Recruiters can't explain to hiring managers why someone ranked top-10

The Solution

Every score is decomposed, justified, and re-tunable on demand:

  • Per-dimension breakdown: skill / experience / domain / location with one-line reasoning each
  • 3-stage matcher mixes hard filters, pgvector recall, and LLM precision
  • Simulated outreach surfaces Interest Score with signals + concerns
  • Recruiter ↔ persona conversations stream live to the UI via SSE
  • Sliders re-weight Match vs. Interest from stored raw scores — no LLM re-call

Core Features

JD Parser

gpt-4o extracts structured fields from any JD: title, seniority, min YoE, must-have skills, nice-to-have, domain, location preference, remote-ok — emitted as JSON.

3-Stage Matcher

Hard SQL filter narrows the pool, pgvector cosine returns top-50 by semantic similarity, and gpt-4o reranks in batches of 10 — emitting per-dimension scores 0–100 with justifications.

Conversation Orchestrator

Recruiter Agent ↔ Candidate Persona Agent run 3-5 turns each, in parallel under Semaphore(5). Every turn streams to the UI via Server-Sent Events.

Judge Agent

Reads each transcript end-to-end and returns an Interest Score plus structured signals[], concerns[], and a one-paragraph reasoning trace.

Tunable Ranker

Default final = 0.6·match + 0.4·interest. UI sliders re-weight from cached raw scores instantly — no LLM re-invocation, no extra cost per slider tweak.

CSV Export

Streaming CSV export of the full ranked shortlist with all per-dimension scores, justifications, and conversation links — ready to drop into an ATS or hiring manager review.

End-to-End Pipeline

A single paste-JD action triggers a six-stage pipeline. Match scoring and outreach run in parallel; the UI streams every turn live so recruiters watch conversations unfold instead of waiting on a spinner.

Setup Guide

1

Spin up Postgres + pgvector

Compose brings up Postgres with the vector extension preloaded:

git clone https://github.com/DandaAkhilReddy/recruiter_agent && cd recruiter_agent && docker compose up -d

Copy .env.example → .env and fill in your AOAI_* values.

2

Backend — migrate, seed, run

FastAPI with Alembic migrations and a synthetic candidate pool:

cd backend && pip install -r requirements.txt && alembic upgrade head && python scripts/seed_candidates.py --count 500 && uvicorn app.main:app --reload --port 8000

Seeds 500 candidates across four interest archetypes (strong / medium / weak / wildcard).

3

Frontend — Next.js dev server

Open the UI and paste a JD:

cd frontend && npm install && npm run dev # http://localhost:3000

Or skip local entirely — the live demo above runs the same pipeline against a seeded Azure Postgres.
